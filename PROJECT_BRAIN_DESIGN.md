**Based on**: Kiro CLI knowledge base structure + research on Claude Code, Aider, Cursor

## Two-Brain Concept

### Brain 1: Coding Brain
**Purpose**: Understand the codebase structure, patterns, and technical stack  
**Index Type**: `fast` (BM25) — because we need exact lookups of functions, classes, files  
**Source**: GreasyNuts backend + frontend codebases

### Brain 2: Functionality Brain  
**Purpose**: Understand what the project does, business domain, requirements  
**Index Type**: `best` (semantic) — because we need conceptual understanding  
**Source**: Specs, product docs, issue descriptions, design decisions

## Indexing Implementation

### Coding Brain (BM25 - Fast)

**Input**: GreasyNuts backend + frontend codebases  
**Process**:
1. Tree-sitter scan → extract functions, classes, methods, imports
2. Tokenize each code chunk (preserve syntax tokens)
3. Build BM25 index:
   - Term frequencies per file
   - IDF scores across corpus
   - Document lengths
4. Store in `bm25_data.json`

**Query**: `find_function("create_purchase_order")` → BM25 keyword match

---

### Functionality Brain (Semantic - Best)

**Input**: Product docs, specs, requirements, decisions  
**Process**:
1. Chunk markdown files by section
2. Generate embeddings using `sentence-transformers/all-minilm-l6-v2`
3. Store vectors in `data.json`

**Query**: "How does the dashboard module handle real-time updates?" → semantic search

3. Scan frontend codebase:
   - Detect: Flutter + Dart

4. Fetch GitHub issue #3:
   - Parse into `sources/functionality-brain/specs/issue-3/requirements.md`
5. Generate `sources/functionality-brain/product.md` from README
6. Index Coding Brain (BM25)
7. Index Functionality Brain (semantic)
8. Write `manifest.yaml`
9. Create agent configs

**Output**: `.project-brain/` ready, both brains indexed

## Query Interface for Agents

### Python API:

```python
from src.brain import ProjectBrain

brain = ProjectBrain.load(".project-brain/manifest.yaml")

# Query coding brain (BM25 keyword search)
results = brain.coding.search("purchase_order create function")
# Returns: [(file_path, chunk, score), ...]

# Query functionality brain (semantic)
results = brain.functionality.search(
    "What are the main features of the dashboard module?",
    top_k=5
)
# Returns: [(content, metadata, similarity_score), ...]

# Get agent context
context = brain.get_agent_context("designer")
# Returns: KB references + source files for designer
```

## Technologies to Use

| Component | Library | Reason |
|-----------|---------|--------|
| **Tree-sitter** | `tree-sitter` + language grammars | AST parsing for structure.md |
| **LSP** | `pygls` (Python LSP server) | Optional deep analysis |
| **BM25** | `rank-bm25` (pip) | Fast keyword search |
| **Embeddings** | `sentence-transformers` | all-minilm-l6-v2 model |
| **Vector Search** | `numpy` + cosine similarity | Simple, no DB needed for 10k files |
| **Chunking** | `langchain` or custom | Smart markdown/code chunking |

## File Limits

- **Max 10,000 files per knowledge base** (Kiro limit)
- **Chunk size**: 512 tokens (balance context vs. granularity)
- **Overlap**: 50 tokens (preserve context across chunks)
