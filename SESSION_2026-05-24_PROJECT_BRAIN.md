### 2. Research: Kiro CLI Knowledge Base Structure ✅

**Web research conducted on:**
- https://kiro.dev/docs/cli/code-intelligence
- https://kiro.dev/docs/cli/steering/
- https://kiro.dev/docs/cli/custom-agents/configuration-reference/

**Key discoveries:**

#### LSP Integration (Code Intelligence)
- `/code init` creates `lsp.json` at project root
- Two-layer architecture:
  - **Tree-sitter** (built-in): 18 languages, symbol search, pattern rewrite
  - **LSP servers** (optional): Deep semantic analysis, go-to-definition, find-references
- Commands: `/code overview`, `/code summary` (generates AGENTS.md)

#### Knowledge Base Internal Structure (from your experience)
```
~/.kiro/knowledge-bases/
├── q_cli_default/                    ← default agent KB
│   ├── contexts.json                 ← manifest of all KBs for this agent
│   └── context-id-1/                 ← specific KB instance
│       ├── data.json                 ← embeddings/chunks (semantic index)
│       └── bm25_data.json            ← BM25 tokens (keyword index)
```

**Two index types:**
1. **`indexType: "best"`** - Semantic search
   - Model: `all-minilm-l6-v2` (384-dimensional embeddings)
   - Best for: documents, research, natural language queries
   - Query: Cosine similarity against stored chunks

2. **`indexType: "fast"`** - BM25 keyword search
   - Algorithm: BM25 (Best Match 25)
   - Best for: logs, config files, large codebases, exact lookups
   - Query: BM25 scoring for keyword relevance

**Key properties:**
- Each agent has isolated KBs (can't access other agents)
- Persistent across sessions
- Supports: code, markdown, CSV, text
- Background indexing for large directories
- Auto-sync with agent resources if configured
- Max 10,000 files per knowledge base

### 3. Designed Project Brain Architecture ✅

**Two-brain concept** (as you requested):

#### Brain 1: Coding Brain
- **Purpose**: Codebase structure, patterns, tech stack
- **Index Type**: BM25 (fast) - for exact function/class lookups
- **Source**: GreasyNuts backend + frontend codebases
- **Query**: `brain.coding.search("purchase_order function")`

#### Brain 2: Functionality Brain
- **Purpose**: Business domain, requirements, what the project does
- **Index Type**: Semantic (best) - for conceptual understanding
- **Source**: Specs, product docs, issue descriptions, design decisions
- **Query**: `brain.functionality.search("how does authentication work")`

**Directory structure:**
```
.project-brain/
├── manifest.yaml                      ← central metadata (THE missing piece in Kiro)
│
├── agents/
│   ├── designer.json                  ← agent configs (Kiro format)
│   ├── planner.json
│   ├── coder.json
│   └── reviewer.json
│
├── knowledge-bases/
│   ├── coding-brain/
│   │   ├── contexts.json              ← KB manifest (Kiro format)
│   │   └── greasynuts-codebase/       ← context-id
│   │       ├── data.json              ← unused (BM25 doesn't need embeddings)
│   │       └── bm25_data.json         ← tokenized codebase
│   │
│   └── functionality-brain/
│       ├── contexts.json
│       └── greasynuts-domain/
│           ├── data.json              ← semantic embeddings
│           └── bm25_data.json         ← unused (semantic doesn't need BM25)
│
└── sources/
    ├── coding-brain/                  ← source files for indexing
    │   ├── structure.md               ← generated by tree-sitter scan
    │   ├── tech-stack.md              ← Python, Flutter, DynamoDB
    │   └── patterns.md                ← code conventions discovered
    │
    └── functionality-brain/
        ├── product.md                 ← what GreasyNuts does
        └── specs/issue-3/
            ├── requirements.md        ← EARS notation
            ├── design.md              ← architecture
            └── tasks.md               ← implementation tasks
```

**Agent runtime query interface:**
```python
from src.brain import ProjectBrain

brain = ProjectBrain.load(".project-brain/manifest.yaml")

# Query coding brain (BM25 keyword search)
results = brain.coding.search("purchase_order create function")

# Query functionality brain (semantic)
results = brain.functionality.search(
    "What are the main features of the dashboard module?",
    top_k=5
)

# LSP queries
definition = brain.lsp.goto_definition("PurchaseOrder", "src/models/order.py")
references = brain.lsp.find_references("create_purchase_order")
```

## Technical Decisions Made

| Decision | Choice | Reason |
|----------|--------|--------|
| **Coding Brain index** | BM25 (fast) | Exact lookups for functions/classes/files |
| **Functionality Brain index** | Semantic (best) | Conceptual understanding of domain/requirements |
| **Embedding model** | all-minilm-l6-v2 | 384 dims, fast inference, good quality (mirrors Kiro) |
| **LSP servers** | pyright + dart analysis | Deep code intelligence (you requested LSP) |
| **Storage location** | `.project-brain/` in AIagentCoding | Outside GreasyNuts repo (as you specified) |
| **Agent access** | Runtime query API | Agents call brain.* dynamically during execution |
| **Manifest format** | YAML | Human-readable, central metadata reference |
| **Chunk size** | 512 tokens | Balance between context and granularity |
| **Max files** | 10,000 per KB | Following Kiro's limit |
